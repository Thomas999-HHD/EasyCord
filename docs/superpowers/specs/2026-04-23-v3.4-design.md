# EasyCord v3.3 + v3.4 — Release Roadmap

This document covers two sequential releases. Each has a single coherent theme and can be planned and shipped independently.

---

# v3.3 — DX + Automation Core

**Theme:** Make bot setup frictionless and invite tracking effortless — zero config where possible, observable everywhere.

**Goal:** Composer auto-intent resolution, `ctx` invite helpers, and `InviteTrackerPlugin` — all three delivered together because they share the same auto-intents foundation.

**Architecture:** Auto-intents resolution lives in `composer.py`. Invite context methods extend `_context_channels.py`. `InviteTrackerPlugin` is a new `SlashGroup` plugin using `bot.db`. No new files except the plugin, its tests, and `easycord/plugins/invite_tracker.py`.

**Tech Stack:** discord.py 2.x, existing `EasyCordDatabase`, existing `SlashGroup` pattern, existing `_context_channels.py` mixin.

See `docs/superpowers/specs/auto_intents.md` for the full auto-intents design spec.

## v3.3 Feature 1: Composer Intents Shortcuts + Auto-Resolution

### Why this is better than discord.py

discord.py:
```python
intents = discord.Intents.default()
intents.members = True
intents.message_content = True
client = discord.Client(intents=intents)
```

EasyCord:
```python
bot = Composer().with_members().with_messages().log().build()
```

Or with full auto-resolution:
```python
bot = Composer().add_group(InviteTrackerPlugin()).build()
# InviteTrackerPlugin.intents auto-merged → members + guild_invites enabled
```

### API

Add to `easycord/composer.py`:

```python
def with_members(self) -> Composer:
    """Enable the privileged Members intent (required for member join/leave events)."""

def with_messages(self) -> Composer:
    """Enable the privileged Message Content intent (required to read message text)."""

def with_reactions(self) -> Composer:
    """Enable the Reactions intent (required for reaction add/remove events)."""

def with_presence(self) -> Composer:
    """Enable the privileged Presences intent (required for online status tracking)."""

def with_voice(self) -> Composer:
    """Enable the Voice States intent (required for voice channel join/leave events)."""

def with_events(self) -> Composer:
    """Enable the Guild Scheduled Events intent."""

def with_invites(self) -> Composer:
    """Enable the Guild Invites intent (required for invite tracking)."""

def with_all(self) -> Composer:
    """Enable all intents (equivalent to discord.Intents.all())."""
```

### Implementation notes

- Each `.with_X()` calls `_ensure_intents()`: returns `self._intents` if set, otherwise initializes to `discord.Intents.default()`.
- `.with_all()` replaces `self._intents` with `discord.Intents.all()` entirely.
- `build()` runs auto-intent collection after all plugins/groups are registered: calls `_collect_plugin_intents()`, merges into `self._intents`, emits `logging.warning` for any privileged intents added automatically.
- Full resolution algorithm: see `auto_intents.md`.

## v3.3 Feature 3: InviteTrackerPlugin

New file: `easycord/plugins/invite_tracker.py`

**What it does:**
1. Snapshots all guild invites on `on_ready` and `on_guild_join`.
2. On `on_member_join`, fetches current invites, diffs against snapshot to identify which code gained a use, records the attribution, updates snapshot.
3. Stores attribution in `bot.db` under key `invite_tracker:{guild_id}:{invite_code}`.
4. Exposes `/invite stats [@user]` and `/invite top [limit=10]` slash commands.

**Intent declaration:**
```python
class InviteTrackerPlugin(SlashGroup, name="invite"):
    intents = {"members", "guild_invites"}
```

**Race condition note:** If two members join within the same second, the snapshot diff may misattribute one. This is inherent to the Discord API; document it in the class docstring.

**Database schema (key-value):**
- Key: `invite_tracker:{guild_id}:{code}` → `{"code": str, "inviter_id": int | None, "uses": int}`
- Key: `invite_tracker:snapshot:{guild_id}` → `{code: uses_count, ...}`

**Plugin is added via `add_group()`, not `add_plugin()`.**

# v3.4 — Event System

**Theme:** First-class scheduled event management with clean lifecycle hooks — developers don't need to read Discord docs to use it.

**Goal:** `ctx.create_event()`, `ctx.get_events()`, `ctx.delete_event()`, and event lifecycle dispatching, all built on top of the auto-intents foundation from v3.3.

**Architecture:** Scheduled event context methods extend `_context_channels.py`. Lifecycle hooks are dispatched via `_bot_events.py`. The `guild_scheduled_events` intent is declared on any plugin that uses lifecycle hooks, resolved automatically via the v3.3 auto-intents system.

**Tech Stack:** discord.py 2.x scheduled events API, existing `_context_channels.py` mixin, existing event dispatch system.

---

## v3.4 Feature 1: Scheduled Event Context Methods

Add to `easycord/_context_channels.py`:

```python
async def create_event(
    self,
    name: str,
    start_time: datetime,
    *,
    end_time: datetime | None = None,
    description: str | None = None,
    channel: discord.abc.GuildChannel | None = None,
    location: str | None = None,
    image: bytes | None = None,
) -> discord.ScheduledEvent:
    """Create a scheduled event in the guild.

    Pass ``channel`` for a voice/stage event; pass ``location`` (a string)
    for an external event. External events require ``end_time``.
    """

async def get_events(self) -> list[discord.ScheduledEvent]:
    """Return all upcoming scheduled events for this guild."""

async def delete_event(self, event: discord.ScheduledEvent | int) -> None:
    """Cancel and delete a scheduled event by object or ID."""
```

### Implementation notes

- Entity type inferred: `channel` → `EntityType.voice` or `EntityType.stage_instance`; `location` → `EntityType.external`; neither → `RuntimeError("Provide channel or location")`.
- `location` without `end_time` → `RuntimeError("External events require end_time")`.
- `channel` + `location` both provided → `RuntimeError`.
- `delete_event` accepts `ScheduledEvent` object or int ID.
- Privacy level always `PrivacyLevel.guild_only`.
- All three methods raise `RuntimeError` if `self.guild is None`.

## v3.4 Feature 2: Event Lifecycle Hooks

Expose clean `@bot.on` bindings for Discord's scheduled event gateway events:

```python
@bot.on("scheduled_event_create")
async def on_event_created(event: discord.ScheduledEvent): ...

@bot.on("scheduled_event_update")
async def on_event_updated(before: discord.ScheduledEvent, after: discord.ScheduledEvent): ...

@bot.on("scheduled_event_delete")
async def on_event_deleted(event: discord.ScheduledEvent): ...

@bot.on("scheduled_event_user_add")
async def on_rsvp(event: discord.ScheduledEvent, user: discord.User): ...
```

These are thin wrappers over discord.py's existing gateway events — no custom dispatch needed, just documentation and ensuring the `guild_scheduled_events` intent is declared when plugins use them.
