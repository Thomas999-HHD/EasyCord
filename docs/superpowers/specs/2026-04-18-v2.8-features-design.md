# v2.8 — Design Spec

**Date:** 2026-04-18  
**Status:** Approved

## Goal

Add fluent UI builders, two infrastructure additions, and in-place codebase quality improvements to the framework. Everything is additive and backwards-compatible.

---

## 1. UI Builders (`easycord/builders/`)

A new `easycord/builders/` package. Four builder classes, each a thin fluent wrapper over the raw discord.py API. All four are exported from `easycord/__init__.py`.

### File map

| File | Class | Returns |
|---|---|---|
| `easycord/builders/embed.py` | `EmbedBuilder` | `discord.Embed` |
| `easycord/builders/button.py` | `ButtonRowBuilder` | `discord.ui.View` |
| `easycord/builders/select.py` | `SelectMenuBuilder` | `discord.ui.View` |
| `easycord/builders/modal.py` | `ModalBuilder` | `dict | None` |
| `easycord/builders/__init__.py` | — | exports all four |

### `EmbedBuilder`

Chainable methods: `.title(text)` `.description(text)` `.field(name, value, inline=True)` `.footer(text)` `.color(color)`. Terminal: `.build()` → `discord.Embed`.

```python
embed = (EmbedBuilder()
    .title("Hello")
    .description("World")
    .field("Score", "100")
    .footer("Bot v2.8")
    .color(discord.Color.green())
    .build())
await ctx.respond(embed=embed)
```

`.title()` is required — `.build()` raises `ValueError("EmbedBuilder requires a title")` if not set. All other methods are optional. `.field()` may be called multiple times.

### `ButtonRowBuilder`

Chainable: `.button(label, custom_id=None, style="primary", url=None)`. Terminal: `.build()` → `discord.ui.View`.

Valid `style` values: `"primary"`, `"secondary"`, `"success"`, `"danger"`. For link buttons, pass `style="link"` and `url="https://..."` instead of `custom_id`. Non-link handlers wired via the existing `@bot.component(custom_id)` router.

```python
view = (ButtonRowBuilder()
    .button("Approve", custom_id="approve", style="success")
    .button("Deny", custom_id="deny", style="danger")
    .build())
await ctx.respond("Approve?", view=view)
```

### `SelectMenuBuilder`

Chainable: `.placeholder(text)` `.option(label, value)`. Terminal: `.build(custom_id)` → `discord.ui.View`.

```python
view = (SelectMenuBuilder()
    .placeholder("Pick one")
    .option("Option A", value="a")
    .option("Option B", value="b")
    .build(custom_id="my_select"))
await ctx.respond("Choose:", view=view)
```

Handler wired via `@bot.component("my_select")`. At least one `.option()` is required — `.build()` raises `ValueError("SelectMenuBuilder requires at least one option")` otherwise.

### `ModalBuilder`

Chainable: `.title(text)` `.field(key, label, *, placeholder=None, required=True)`. Terminal: `await .send(ctx)` → `dict[str, str] | None`.

```python
result = await (ModalBuilder()
    .title("Feedback")
    .field("reason", "Why?", placeholder="Tell us…")
    .field("detail", "Details", required=False)
    .send(ctx))
if result:
    reason = result["reason"]
```

Delegates to `ctx.ask_form()`. Returns `None` on timeout or dismiss. `.title()` is required — `.send()` raises `ValueError("ModalBuilder requires a title")` if not set.

## 2. Infrastructure

### `bot.reload_plugin(name: str)`

Unloads a loaded plugin by class name and re-adds it from scratch.

```python
await bot.reload_plugin("TagsPlugin")
```

Implementation in `easycord/_bot_plugins.py`:
1. Find the loaded plugin whose class name matches `name`.
2. Call `on_unload()` on the instance directly (preserves constructor args and state).
3. Call `on_load()` on the same instance again.

Raises `ValueError("No plugin named '{name}' is loaded")` if not found.

### `@slash(aliases=[...])`

Registers the same handler under one or more additional command names. Aliases inherit all options from the parent (guild_id, cooldown, permissions, etc.).

```python
@bot.slash(description="Show help", aliases=["?", "commands"])
async def help(ctx):
    ...
```

Implementation in `easycord/_bot_commands.py`: after registering the primary command, iterate `aliases` and call `bot.tree.add_command` for each, reusing the same callback.

## 3. Codebase improvements

Applied in-place across files touched by this release — no standalone cleanup pass.

- **Type hints:** full annotations on all new code; existing functions in touched files get return types and `str | None` style unions.
- **Error messages:** three specific guard cases (see below); no speculative guarding.
- **No unrelated cleanup:** only improve what we're already editing.

### Error message guards

| Location | Condition | Exception |
|---|---|---|
| `ctx.guild_id` / `ctx.guild` property | interaction has no guild | `RuntimeError("This command is guild-only — ctx.guild_id is not available in DMs")` |
| `bot.reload_plugin(name)` | name not found in loaded plugins | `ValueError("No plugin named '{name}' is loaded")` |
| `EmbedBuilder.build()` | `.title()` not called | `ValueError("EmbedBuilder requires a title")` |
| `SelectMenuBuilder.build()` | no `.option()` called | `ValueError("SelectMenuBuilder requires at least one option")` |
| `ModalBuilder.send()` | `.title()` not called | `ValueError("ModalBuilder requires a title")` |

## 4. Tests

Each builder gets a dedicated test file. Infrastructure additions get tests appended to existing test files.

| Test file | Covers |
|---|---|
| `tests/test_embed_builder.py` | `EmbedBuilder` — all methods, `.build()` output, missing title guard |
| `tests/test_button_builder.py` | `ButtonRowBuilder` — button count, styles, view type |
| `tests/test_select_builder.py` | `SelectMenuBuilder` — options, placeholder, empty guard |
| `tests/test_modal_builder.py` | `ModalBuilder` — delegates to `ask_form`, returns dict/None, missing title guard |
| `tests/test_bot.py` | append: `reload_plugin` happy path + unknown name error |
| `tests/test_bot.py` | append: `aliases` registers extra commands |
